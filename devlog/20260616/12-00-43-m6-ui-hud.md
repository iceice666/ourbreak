## Technical notes

- New dependency: `com.simsilica:lemur:1.16.0` (same author as our Zay-ES). Its
  only transitive deps ‚Äî jme3-core, guava, slf4j-api ‚Äî are already present, and
  it pulls no Groovy. `GuiGlobals.initialize` runs in `OurcraftGame`; styling is
  plain Java (no glass/Groovy style), keeping the footprint to Lemur itself.
- UI is runtime-visual, so it was verified by launching the app in WSLg; only
  `HudText` is unit-tested. Headless suite green incl. `HudTextTest`.

---

## Decisions made

- **Decision**: Lemur with default Java styling, no glass style.
  **Reason**: clickable, readable widgets without dragging in Groovy.
  **Alternatives considered**: `BaseStyles.loadGlassStyle()` (rejected ‚Äî Groovy).

- **Decision**: `HudState` is a thin view; `HudText` holds the logic.
  **Reason**: keeps formatting unit-testable with no jME dependency.
