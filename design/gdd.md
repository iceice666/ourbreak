# 核心玩法概述
遊戲名稱: ourcraft
遊玩時間: 無限生存模式，一局長度取決於撐過的回合數（刷新個人最高紀錄）
遊戲類型: 第一人稱動作遊戲
![[democoncept.png]]

## 角色固定
>考慮到開發時間，目前角色為固定，如時間其實充裕，則玩家可以扮演建造者
- **玩家**：永遠是破壞者
- **NPC**：永遠是建造者

## Round 結構（無限生存模式）
無限個 round，難度逐回合遞增。每個 round 分兩個階段：

| 階段   | 執行者 | 規則                              |
| ---- | --- | ------------------------------- |
| 建造階段 | NPC | 自動在吉祥物周圍放置方塊（牆越來越厚、越來越難），方塊放完結束 |
| 攻擊階段 | 玩家  | 1 分鐘時間限制，要在時間內清光整面牆             |

- 建造階段：玩家無法操作
- 攻擊階段：NPC 靜止不動

## 勝負條件（生存 / 計分）
- **存活並前進**：在攻擊階段時間內清光所有建築 → 視為「撐過這一 round」，進入下一個（更難的）round。**沒有「贏」的終局**。
- **遊戲結束（Game Over）**：任一 round 的攻擊計時器歸零時仍有建築存在 → 立即結束。
- **分數**：撐到第幾個 round（結束畫面顯示 `Reached Round N`）。目標是刷新自己的最高紀錄。

## 世界觀
Clawd 跟 Openclaw 是海灘上的兩個鄰居，兩個都想在沙灘上蓋自己的夢幻沙堡。
有一天 Openclaw 不小心把 Clawd 辛苦蓋的沙堡踩爛了，從此兩個就槓上了，每天在沙灘上互相拆對方的堡壘。

# 方塊（NPC 使用）
NPC 每個建造階段自動放置方塊保護吉祥物。
每 round 可用方塊數量逐回合遞增、**無上限一直變大**：第 1–4 round 是教學斜坡（16 / 24 / 32 / 40，逐步引入機制），第 5 round 起進入「無限引擎」——以**要求清除速率**為難度主軸：速率 `ρ(r)=1.20−0.40×0.85^(r−5)`（漸近逼近 1.20、增量逐回合縮小不暴衝），攻擊時間 `T(r)=60+2×(r−5)` 秒緩慢成長，方塊數 `= round(ρ×T)`（R5 48 / R6 53 / R10 72 / R15 90 / R20 105 / R30 131…，越來越大但永遠清得完）。方塊在吉祥物周圍排成**立體同心城牆**：每圈先鋪滿地面層再往上疊高（牆高 3 格），不夠才往外擴一圈。

| 方塊                   | 耐久  | 功能              |
| -------------------- | --- | --------------- |
| Sand Block（沙塊）       | 1 下 | 無特殊，便宜量多，適合快速堆牆 |
| Coral Block（珊瑚塊）     | 2 下 | **再生/蔓延**：存活時每 7 秒在相鄰空洞長出一塊新珊瑚 → 牆會自己癒合長回來，留著清不完（封頂＝攻擊起始時那面牆的範圍）。次要：玩家靠近 1.5 格內也會被減速 |
| Shell Block（貝殼塊）     | 1 下 | 被劍/無人機打 → 分裂成 2 個貝殼；用槍乾淨清掉 |
| Rock Block（礁石塊）      | 4 下 | 純高耐久，沒有特效       |
| Jellyfish Block（水母塊） | 1 下 | 用**無人機**炸死 → 玩家中毒 +5 秒（上限 10 秒）；用槍/劍乾淨清掉不中毒。中毒時不管拿什麼武器，全場真方塊都亂跳彩虹色，分不出方塊種類，並有一條會遞減的中毒條 UI |

# 武器（玩家使用）
玩家在攻擊階段可自由切換武器。
不同武器剋不同方塊，是勝負關鍵。

| 武器         | 攻擊方式          | 剋                                               | 被剋                                            |
| ---------- | ------------- | ----------------------------------------------- | --------------------------------------------- |
| Sword（劍）   | 近戰，一揮可破多個相鄰方塊 | Sand Block（橫掃一排）                                | Shell Block（會分裂變多）、Coral Block（牆一直長回來、近身又被減速）     |
| Gun（槍）     | 遠距單體射擊        | Coral Block（一發點掉再生源，斬草除根）、Jellyfish Block（清掉不中毒） | Rock Block（單體傷害對高耐久效率差）                       |
| Drone（無人機） | 範圍轟炸，需幾秒操控時間  | Rock Block（範圍傷害無視高耐久）、Sand Block（大範圍清除）         | Jellyfish Block（炸死會中毒，全場變彩虹分不出方塊）、Shell Block（炸到會分裂雪崩） |

## Mechanics

#### 計時器
| 項目 | 數值 |
| ---- | ---- |
| 攻擊階段時間限制 | 60 秒（第 1–5 round）；第 5 round 起每回合 +2 秒、無上限（`60+2×(round−5)`） |
| 總 round 數 | 無限（生存模式，難度漸近遞增、永不暴衝也永不封頂） |

#### 方塊數值
| 方塊 | 耐久（下） | 特效範圍 / 數值 |
| ---- | --------- | -------------- |
| Sand Block | 1 | — |
| Coral Block | 2 | **再生間隔 7 秒**：每個活珊瑚每 7 秒在相鄰（6 面）空洞長一塊新珊瑚，只能在攻擊起始的牆範圍內補（封頂＝長回整面牆）。次要效果：減速範圍 1.5 格、移動速度 ×0.5 |
| Shell Block | 1 | 被劍/無人機打 → 分裂成 2 個新貝殼（無上限）；用槍乾淨清掉。玩家無血量 |
| Rock Block | 4 | — |
| Jellyfish Block | 1 | 無人機炸死 → 中毒 +5 秒（上限 10 秒）；槍/劍清掉不中毒。中毒時全場真方塊亂跳彩虹色（每 ~0.12 秒換色），中毒條滿格 = 10 秒 |

每 round NPC 可放方塊數量（無上限遞增）：R1–4 `16+(r-1)×8`（16/24/32/40）；R5+ `round(ρ(r)×T(r))`,其中 `ρ(r)=1.20−0.40×0.85^(r−5)`、`T(r)=60+2×(r−5)` → R5 48 / R6 53 / R8 63 / R10 72 / R15 90 / R20 105 / R30 131。

**難度設計理念（數學＋心理學）**：以「要求清除速率 ρ = 方塊數 ÷ 時間」當難度主軸。ρ 漸近逼近 1.20（人類極限之下）但**永遠到不了** → 任何牆理論上都清得完,玩家是輸在自己技巧用盡而非遊戲耍賴。增量逐回合 ×0.85 收斂（像調和級數:發散但放緩）→ 難度**無限上升卻永不暴衝**,維持心流通道(挑戰略高於技巧)。牆同時越來越大(視覺升級＋珊瑚變多→再生壓力自然放大)避免單調。

#### 武器數值
| 武器 | 攻擊距離 | 範圍 | 基礎傷害 |
| ---- | ------- | ---- | ------- |
| Sword | 2 格 | 3x1 橫掃 | 1.0 |
| Gun | 20 格 | 單體 | 8.0（一槍秒殺任何單一方塊） |
| Drone | — | 3x3 爆炸 | 1.0 |

> 實際傷害 = 基礎傷害 × 剋制倍率（剋 ×2.0、被剋 ×0.5、普通 ×1.0）。各武器/方塊的命中次數見 `design/tdd.md`。
